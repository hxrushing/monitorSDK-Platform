# SDK测试页面实现总结

## ✅ 已完成

### 测试页面功能

1. **SDK配置区域**
   - 可配置项目ID和上报地址
   - SDK初始化按钮
   - 初始化状态显示

2. **阶段1 (P0) - 核心能力测试**
   - ✅ API接口测试：track, trackError, trackPage, trackHttp, trackPerf, flush, setUser
   - ✅ 批量上报测试：批量发送多个事件，检查队列状态

3. **阶段2 (P1) - 探针测试**
   - ✅ 错误探针测试：触发JS错误、Promise拒绝、资源加载错误

4. **测试结果展示**
   - 实时显示测试结果
   - 按类型分类（success/error/info）
   - 显示时间戳

5. **队列状态监控**
   - 检查离线缓存中的事件数量
   - 显示在线/离线状态

## 📝 使用说明

1. 访问路径：`/app/sdk-test`
2. 配置项目ID和上报地址（默认为localhost:3000）
3. 点击"初始化SDK"
4. 依次测试各个功能
5. 查看测试结果和网络请求

## 🔍 验证要点

- SDK初始化成功
- 各个API能正常调用
- 事件能正确上报到服务器（查看Network面板）
- 错误探针能自动捕获并上报错误
- 批量上报机制正常工作
- 队列状态显示正确

## 📌 注意事项

1. 确保后端服务运行在配置的上报地址
2. 某些测试会触发真实的错误（JS错误、Promise拒绝等）
3. 可以切换到离线模式测试离线缓存功能
4. 查看浏览器控制台和网络面板以获取更详细的信息

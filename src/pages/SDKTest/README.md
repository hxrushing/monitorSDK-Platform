# SDK测试页面使用说明

## 功能说明

此测试页面用于验证可插拔探针SDK各个功能的正确性。

## 测试内容

### 阶段1 (P0) - 核心能力测试

1. **API接口测试**
   - `track()` - 自定义事件跟踪
   - `trackError()` - 错误跟踪
   - `trackPage()` - 页面浏览跟踪
   - `trackHttp()` - HTTP请求跟踪
   - `trackPerf()` - 性能指标跟踪
   - `flush()` - 手动刷新队列
   - `setUser()` - 设置用户ID

2. **批量上报测试**
   - 批量发送多个事件
   - 检查队列状态
   - 验证批量上报机制

### 阶段2 (P1) - 探针测试

1. **错误探针测试**
   - 触发JS错误
   - 触发Promise拒绝
   - 触发资源加载错误

## 使用步骤

1. 配置项目ID和上报地址
2. 点击"初始化SDK"
3. 依次测试各个功能
4. 打开浏览器开发者工具查看网络请求
5. 查看测试结果列表

## 验证要点

- ✅ SDK初始化成功
- ✅ 各个API调用正常
- ✅ 事件能正确上报到服务器
- ✅ 错误探针能自动捕获错误
- ✅ 批量上报机制正常工作
- ✅ 离线缓存功能正常（可切换到离线模式测试）

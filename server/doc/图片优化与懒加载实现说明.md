# 图片优化与懒加载实现说明

## 改动点
- 文件：`src/components/OptimizedImage.tsx`
  - 支持 IntersectionObserver 懒加载，进入视口才加载真实图源。
  - 支持 `placeholder` 占位（低清晰度或透明占位）。
  - 支持 `srcSet` / `sizes`，便于响应式图片按需加载合适清晰度。
  - 仍保留 `loading='lazy'`、错误回退、fetchpriority 设置。

## 使用示例
```tsx
<OptimizedImage
  src="/assets/banner@2x.jpg"
  placeholder="/assets/banner-blur.jpg"
  alt="banner"
  width={1200}
  height={400}
  srcSet="/assets/banner@1x.jpg 1x, /assets/banner@2x.jpg 2x"
  sizes="(max-width: 768px) 100vw, 1200px"
/>
```

## 兼容性与注意事项
- IntersectionObserver 已在现代浏览器普遍支持；老旧浏览器可按需引入 polyfill。
- 提前加载距离：`rootMargin: 200px 0px`，可按需调整。
- 若页面需要 LCP 图（首屏主图），设置 `isLCP` 或 `loading='eager'`，组件会直接加载并设 fetchpriority 为 high。
